Safety Gate

Discord admin permission is not the final key.

CLINX adds its own runtime trust system so destructive power is separated from normal administrative access.

Tier 0

Public info only.

Tier 1

Safe admin actions.

Tier 4

Owner-only governance.

Owner

The actual guild owner is always trusted and can approve or execute protected actions.

Trusted Admin

Runs Tier 2 and Tier 3 directly, but never bypasses owner-only governance.

Untrusted Admin

Can request high-impact actions, but CLINX pauses the run for owner approval.

Deletion Lock

Roles and channels are protected from normal admin deletion unless an approved load is meant to change them.

Single Pending Request

One protected approval queue per guild keeps decision paths clean and prevents spam collisions.

Central Dispatch Pacing

Protected bulk writes stay on the throttled path instead of using special fast lanes.

Workflows

Three common routes cover most restore pressure.

Choose the route by blast radius, then let CLINX narrow or gate the action set.

Merge Route

Patch without deletes

Use load with delete toggles off when you want the backup to add and update, not wipe.

Missing Route

Recover only the gaps

Restore missing structures when the incident is partial and the live server is still usable.

Rebuild Route

Mass restore structure fast

Use backup loads or pasted layouts when a server needs a larger structural rebuild.

Backup load behavior

Merge-only is treated as Tier 2. Delete-enabled loads move to Tier 3 and can require explicit owner approval.

Protected planner

The private planner stays with the requester, but approved execution uses the exact frozen toggles selected at request time.

Owner Approval

Protected requests become visible cards, not silent actions.

When an untrusted admin requests a Tier 2 or Tier 3 command, CLINX turns it into an owner decision in the same channel.

Step 01

Request

Untrusted admin starts a protected action.

Step 02

Card Posted

Owner is tagged in the same channel where the request happened.

Step 03

Payload Frozen

Route, selected lanes, and planned changes are locked before approval.

Step 04

Approve or Deny

Approve runs it. Deny or expiry leaves the server untouched.

Approval card rules

Same channel Owner-only buttons 2 minute expiry Denied and expired states Running status updates
