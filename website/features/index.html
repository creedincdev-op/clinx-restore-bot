Safety Gate

High-impact commands follow trust rules at runtime.

Discord admin permission alone is not the final key. CLINX adds its own safety layer on top.

Tier 0

Public info only.

Tier 1

Safe admin actions run directly.

Tier 2

High-impact but non-destructive actions need approval when untrusted.

Tier 3

Destructive actions stay behind stronger approval gates.

Tier 4

Owner-only governance commands.

Owner

The actual guild owner is fully trusted and can run all protected commands, including governance.

Trusted Admin

Trusted admins can run Tier 2 and Tier 3 directly, but Tier 4 still stays owner-only.

Untrusted Admin

Untrusted admins can request protected actions, but the owner decides with approve or deny buttons.

Deletion Lock

Channels and roles stay protected from normal admin deletion unless an approved load intentionally changes them.

One Pending Request

Only one protected request can wait per guild, which keeps approvals clear and stops spam chains.

Central Pacing

Bulk write actions stay on the throttled dispatch path so fast restore runs do not bypass safety pacing.

Owner Approval

Protected actions become a live approval card instead of a silent execute.

The request is frozen at creation time, posted in the same channel, and only the server owner can decide.

Step 01

Request

An untrusted admin starts a Tier 2 or Tier 3 action.

Step 02

Owner Ping

CLINX posts a public approval card in the same channel and tags the actual guild owner.

Step 03

Frozen Plan

The selected route, toggles, and planned changes are locked before approval happens.

Step 04

Run or Deny

Approve starts the action and updates status live. Deny or expiry does nothing.

Approval card rules

Same channel Owner-only buttons 2 minute expiry Running / completed / failed Denied / expired states
Command Access

Command groups stay readable because access follows the tier model.

Trusted admins bypass approval for Tier 2 and Tier 3. The owner bypasses all gates. Tier 4 stays owner-only.

Tier 0 - Public Info

/help /invite

Tier 1 - Safe Admin

/backup create /backup list /backup delete /backupcreate /backuplist /panel suggestion export commands

Tier 2 - Approval When Untrusted

Non-destructive high-impact actions.

Tier 3 - Destructive

Delete-enabled loads and cleanup paths.

/backup load with delete toggles /backupload destructive mode /cleantoday

Tier 4 - Owner Only
