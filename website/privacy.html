Privacy Policy

CLINX keeps data handling narrow

CLINX is built around backup and restore workflows. The service is intended to process only the information required to execute those workflows and support admins during recovery operations.

Minimum necessary data No data sale

Data CLINX May Process

  • Server structure snapshots such as roles, channels, categories, and related configuration values.
  • Backup identifiers and restore workflow metadata required to complete import or load operations.
  • Logs or screenshots voluntarily shared by admins during support requests.

How Data Is Used

  • To create backups, compare restore targets, and execute selected load actions.
  • To diagnose failed commands, import issues, or bot configuration problems.
  • To improve workflow safety, visibility, and product support.

Retention and Control

Backup snapshots may be stored locally as part of the bot's restore workflow. Admins retain operational control over those files and can remove local backup data from storage when it is no longer needed.

What CLINX Does Not Do

  • CLINX does not sell user data.
  • CLINX is not intended to collect more information than the workflow requires.
  • Support data is not collected unless you choose to share it.
